DESCRIPTION  AND  REQUIREMENTS  
The  project  consists  of  a  web  application  that  must  satisfy  a  number  of  technical,  stylistic,  and

The  web  application  must  use  the  technologies  illustrated  and  tested  during  the  course.  
Logistical  Requirements  
●  The  project  must  be  carried  out  individually.  ●  A  submission  that  is  partially  or  entirely  shared  with  another  student  enrolled  in  the

 ●  The  project  must  be  submitted  according  to  the  deadlines  indicated  on  the  last  page  of

 ●  Late  submissions  are  neither  allowed  nor  acceptable.  
Technical  requirements  
The  web  application  must  comply  with  the  following  technical  requirements:  
 


  ●  Use  of  HTML5  and  CSS3,  relying,  if  necessary,  on  external  frameworks  such  as  Bootstrap,

 ●  Use  of  Flask  and  SQLite,  as  a  relational  database,  for  the  back  end.  ●  Use  of  Flask-Login  for  authentication  management.  
In  addition:  
●  The  web  application  must  have  a  clearly  deﬁned  target  device  type,  chosen  by  the  
students:
 
desktop,
 
mobile,
 
or

 ●  Any  forms  used  in  the  project  must  be  validated  both  on  the  front  end  and  on  the  back

 ●  The  submitted  project  must  be  fully  testable  by  the  instructor  and  must  work  on  the  latest

111+,

Firefox,
 
110+.
 ●  The  source  code  must  be  well  written  and  include  appropriate  comments  where

 ●  All  the  technologies  listed  above  must  be  integrated  cohesively  and  consistently  within  a  
single
 
web
 
application.
 
Required:  
●  Deploy  the  web  application  on  PythonAnywhere:  https://www.pythonanywhere.com/  
Stylistic  Requirements  
The  web  application  must  comply  with  the  following  stylistic  requirements:  
●  Use  HTML  tags  semantically,  for  example,  not  everything  should  be  a  <div>.  ●  Do  not  use  deprecated  HTML  tags.  ●  Do  not  use  inline  CSS  declarations;  CSS  rules  must  be  kept  separate  from  the  semantic  
structure
 
of
 
the
 
HTML.
 
In  addition,  the  web  application  must  be  suﬃciently  usable. 
Description:  Functional  requirements  
The  goal  is  to  create  a  web  application  for  managing  Free  Walking  Tours  in  a  city  chosen  by  the

The  web  application  must  support  two  types  of  registered  users:  guides  and  participants .  


  Guides  can  create  and  manage  the  Free  Walking  Tours  oﬀered  on  the  platform.  Before  acting  as

The

A  participant  cannot  become  a  guide,  and  a  guide  cannot  make  reservations  as  a  participant.  
Each  Free  Walking  Tour  is  created  and  managed  by  a  single  guide  and  must  include  the  following
